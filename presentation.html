The Patient · Seven reasons this needed to change
Seven Reasons
This Needed to Change
Pinned Bug
GET /albums/{id} → 200 + null
Missing album returns 200 with a null body. Not 404. Pinned in characterization tests before we touched anything. Corrected cleanly in the new service.
Dead Field in API
albumId — always null
Every response carries albumId. It is never set. Never read. The ACL contract test ensures it never leaks into the new service.
Wrong Type
releaseYear: "1984" (String)
A year is an integer. Typed wrong from day one. Contract tests now enforce typeof releaseYear === 'number' permanently.
Security
Kill-switch endpoints in prod
/errors/kill calls System.exit(1). Anyone with the URL terminates the process. No auth. No rate-limit. No joke.
No Safety Net
1 test total
contextLoads(). That is it. We added 29-album characterization tests — bugs pinned and all — before touching a single line of logic.
EOL Stack
Spring Boot 2.4 · AngularJS 1.2
Both EOL since 2021. Active CVEs in AngularJS, Bootstrap 3, and jQuery 2. No patch path exists. Replace, do not upgrade.
The Strategy · ADR-001
Why Strangler Fig,
Not a Rewrite
Why Not Rewrite
• Zero behavioral tests → rewrite risks silent breakage
• 4 DB backends via profiles → unknown prod dependencies
• Rewrite proves nothing; Strangler Fig proves everything
• Rollback is trivial at every seam
What We Chose NOT to Do
• No Spring Boot 3 upgrade in-place (javax→jakarta touches every file)
• No shared DB between monolith and new service
• No gradual releaseYear String→int in-place migration

Deterministic ACL enforcement
A Python script wired to every Write/Edit in services/. Regex-scans content for monolith field leakage. Blocks the tool call with exit code 2 before the file is written. Not a suggestion. Not a prompt. A hard stop inside the execution pipeline.

ADR-003 · Hook vs Prompt — Why We Use Both
Hook: binary invariant, no exceptions, cannot be reasoned around by the model.
CLAUDE.md: directional preference requiring judgment, where a hook would over-block legitimate writes.
